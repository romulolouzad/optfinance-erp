# Design System Document: The Precision Architect

This design system establishes a high-end, editorial visual identity for a sophisticated B2B ERP environment. Moving away from the cluttered, "spreadsheet-style" legacy of enterprise software, this system adopts an architectural approach—prioritizing depth, intentional white space, and a curated color hierarchy to transform complex data into an authoritative and premium user experience.

---

## 1. Overview & Creative North Star
**Creative North Star: The Precision Architect**
The system is built on the principle of "Structural Clarity." Instead of containing data within rigid boxes and heavy lines, we treat the UI as a series of layered planes. We break the "template" look by using high-contrast typography scales and intentional asymmetry in layouts. The goal is to make the user feel like they are navigating a high-end digital workspace where every pixel serves a functional purpose, wrapped in a signature, premium aesthetic.

## 2. Colors & Surface Logic

### Color Palette (Material Token Mapping)
*   **Primary (The Accent):** `primary_container` (#F97316) – Used for critical actions and brand presence.
*   **Neutral Base:** `surface` (#F3F4F6) – The foundation of the main content area.
*   **Deep Contrast:** `inverse_surface` (#111827) – Reserved for the Sidebar and navigation, providing an authoritative anchor.
*   **Surface Hierarchy:** `surface_container_lowest` (#FFFFFF) through `surface_container_highest` (#DCE2F7).

### The "No-Line" Rule
To achieve a high-end editorial feel, designers are **strictly prohibited** from using 1px solid borders for sectioning or layout containment. 
*   **Structural Separation:** Boundaries must be defined through background color shifts. For example, a card (`surface_container_lowest`) must sit on a main background (`surface`) to define its edge. 
*   **Nesting Depth:** Treat the UI as physical layers. An inner dashboard widget should use a slightly higher tier (e.g., `surface_container_high`) than its parent container to signify importance.

### The "Glass & Gradient" Rule
Flat colors can feel "out-of-the-box." To add "soul" and professional polish:
*   **Primary Buttons:** Use a subtle vertical gradient from `primary` (#9D4300) at the bottom to `primary_container` (#F97316) at the top.
*   **Floating Elements:** For user profile dropdowns or tooltips, apply **Glassmorphism**. Use a semi-transparent `surface_container_lowest` with a 12px `backdrop-blur` to allow the background context to bleed through.

## 3. Typography: Editorial Authority
The typography scale utilizes **Inter** to balance technical precision with modern readability.

*   **Display & Headlines:** Use `display-md` and `headline-lg` for dashboard summaries. These must use `on_surface` (#111827) with a font-weight of **Bold (700)** and a tight letter-spacing (-0.02em) to mimic high-end financial journals.
*   **Title & Body:** `title-md` serves as the primary navigation and card header font. `body-md` uses #374151 to reduce eye strain during long-form data entry.
*   **Labels:** Use `label-md` for status badges and micro-copy. These should always be uppercase with increased letter-spacing (+0.05em) to differentiate them from functional data.

## 4. Elevation & Depth

### The Layering Principle
Depth is achieved via **Tonal Layering** rather than traditional drop shadows.
1.  **Level 0 (Base):** `surface` (#F3F4F6) – The main app canvas.
2.  **Level 1 (Cards):** `surface_container_lowest` (#FFFFFF) – The primary data container.
3.  **Level 2 (In-Card Elements):** `surface_container_low` (#F1F3FF) – Search bars or secondary lists within a card.

### Ambient Shadows
When a component must "float" (e.g., a primary CTA or a modal), use an **Ambient Shadow**:
*   **Blur:** 24px - 40px.
*   **Opacity:** 4% - 8% of `on_surface`.
*   **Color Tint:** Shadows should never be pure black; they must be tinted with the surface color to feel integrated into the environment.

### The "Ghost Border" Fallback
If a border is required for accessibility, it must be a **Ghost Border**: Use `outline_variant` at **15% opacity**. This provides a hint of structure without interrupting the visual flow.

## 5. Components

### The Sidebar
*   **Background:** `inverse_surface` (#111827).
*   **Width:** 220px.
*   **Typography:** White labels with icons. Active states must use a vertical 4px "pill" indicator in `primary_container` (#F97316) on the left edge.

### Buttons
*   **Primary:** `primary_container` fill with `on_primary` (White) text. **8px (0.5rem) rounded corners.**
*   **Secondary:** Ghost Border style—`surface_container_lowest` background with a subtle `outline` (#8C7164) and `primary` text.
*   **Interactive State:** On hover, primary buttons should shift toward `primary_fixed_dim` for a soft glow effect.

### Data Tables (The "Precision" Grid)
*   **Header:** `surface_container_low` (#F9FAFB) with `label-md` bold text.
*   **Rows:** No horizontal or vertical lines. Use `surface_container_lowest` as the row background.
*   **Hover State:** `row_hover` (#FFF7ED). This subtle orange-tinted hover provides immediate feedback without the harshness of a dark gray highlight.

### Status Badges
*   **Shape:** Full pill (`rounded-full`).
*   **Style:** Soft-tinted backgrounds (e.g., an "Active" badge uses 10% opacity of the accent color with the full-strength accent color for the text).

---

## 6. Do's and Don'ts

### Do
*   **DO** use whitespace as a structural element. If elements feel too close, increase the spacing rather than adding a divider line.
*   **DO** use the typography hierarchy to lead the user's eye—make the most important data point the largest and boldest element on the page.
*   **DO** use `surface_container` tiers to create logical grouping of information.

### Don't
*   **DON'T** use high-contrast 1px borders. This is the fastest way to make a premium system look like a generic template.
*   **DON'T** use pure black for text or shadows. Use the `on_surface` or `on_background` tokens to maintain tonal harmony.
*   **DON'T** clutter the sidebar. Use "The Precision Architect" approach: keep top-level nav clean and use sub-menus for deep ERP functionality.
